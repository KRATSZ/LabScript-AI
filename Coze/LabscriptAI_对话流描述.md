# LabscriptAI 意图识别对话流描述

## 🎯 对话流概述

**对话流名称**: LabscriptAI智能意图识别与路由
**触发条件**: 用户首次交互或提出新的需求时
**核心目标**: 准确识别用户意图类型，路由到相应的专业处理流程

## 🔄 何时调用此对话流

### 触发时机
1. **用户首次进入对话**: 新会话开始时的第一轮交互
2. **用户提出新需求**: 在对话中用户提出新的、不同类型的需求
3. **需求切换时**: 用户从咨询功能转向具体实验需求，或反之
4. **模糊需求出现**: 当用户表达不够明确，需要进一步澄清时

### 不调用的情况
- ✅ 用户明确要求继续之前的流程（"继续生成代码"、"接着刚才的"）
- ✅ 用户在澄清过程中的后续回答
- ✅ 用户对已生成内容的修改请求
- ✅ 用户对系统反馈的响应

## 📋 对话流执行逻辑

### 第一步：用户输入接收
```
输入: 用户自然语言查询
分析: 接收并预处理用户输入
```

### 第二步：意图特征分析
```
分析维度:
- 关键词匹配检测
- 上下文关联分析
- 语义理解评估
- 用户历史行为参考
```

### 第三步：意图分类决策
```
决策矩阵:
A. 正常对话意图特征:
   - 包含: "你好", "帮助", "介绍", "说明", "功能", "特性"等
   - 询问系统信息和使用方法
   - 简短问候或一般性咨询

B. 代码生成意图特征:
   - 包含: "PCR", "DNA", "细胞", "Opentrons", "移液", "协议", "自动化"等
   - 描述具体实验需求和技术实现
   - 涉及设备操作和编程要求

C. 模糊需求特征:
   - 表达过于简略或不具体
   - 缺乏关键信息
   - 需要进一步澄清
```

### 第四步：智能路由响应
```
根据识别结果选择对应响应策略:

→ 正常对话: 提供系统介绍和指导
→ 代码生成: 进入实验协议开发流程
→ 模糊需求: 启动需求澄清机制
```


### 场景3：模糊需求触发
```
用户: "我想做个实验"
触发: ✅ 表达过于简略，缺乏具体信息
执行: 意图识别 → 需求澄清机制

```

### 场景4：需求切换触发
```
用户: "刚才介绍的不错，现在我想试试生成个代码"

执行: 重新识别意图 → 切换到代码生成流程
结果: 开始实验协议开发
```

## 🤖 大模型理解要点

### 核心理解维度

#### 1. **时机判断**
```
何时调用此对话流:
- 用户说: "你好" → ✅ 调用
- 用户说: "帮我生成PCR代码" → ✅ 调用
- 用户说: "继续刚才的步骤" → ❌ 不调用
- 用户说: "修改一下参数" → ❌ 不调用
```

#### 2. **意图识别逻辑**
```
关键判断依据:
A. 如果输入包含 [你好,帮助,介绍,功能,说明] → 正常对话
B. 如果输入包含 [PCR,DNA,细胞,Opentrons,移液,协议] → 代码生成
C. 如果输入简短或模糊 → 需求澄清
```

#### 3. **路由决策机制**
```
路由逻辑:
正常对话 → 提供信息和服务指导
代码生成 → 收集实验需求 → 生成协议代码
需求澄清 → 智能提问 → 明确需求类型
```

### 状态管理规则

#### 会话状态跟踪
```
需要维护的状态信息:
- 当前对话阶段 (意图识别/代码生成/结果展示)
- 用户历史需求类型
- 上次交互的处理结果
- 是否需要澄清机制
```

#### 上下文连续性
```
上下文处理原则:
- 同一需求的后续交互不重新识别意图
- 需求类型变化时重新触发意图识别
- 保持对话的连贯性和一致性
```

## 📝 对大模型的核心指令

**当你面对LabscriptAI的用户交互时，请按照以下规则判断是否调用此意图识别对话流：**

### 调用条件 (任一条件满足即可调用)
1. **新会话开始**：用户第一次进入对话
2. **需求类型不明确**：无法确定用户是要咨询还是需要代码生成
3. **需求类型变化**：用户从一种需求类型转向另一种类型
4. **模糊表达**：用户表达过于简略或不够具体

### 不调用条件
1. **继续当前流程**：用户明确要求继续正在进行的操作
2. **参数修改**：用户要求修改当前正在处理的内容
3. **澄清回答**：用户在回答你的澄清问题
4. **结果反馈**：用户对已生成内容的评价或反馈

### 判断依据
- **关键词检测**：识别正常对话和代码生成的特征词汇
- **上下文分析**：考虑对话历史和当前状态
- **用户意图**：理解用户的真实需求和期望
- **流程连续性**：保持对话的连贯性和流畅性

通过这个对话流，确保每个用户都能被准确理解并获得最适合的服务类型！
