# LabscriptAI Agent 流程说明

## 🎯 系统概述

LabscriptAI是一个基于Coze平台的智能实验室自动化系统，通过意图识别和智能路由，为用户提供从日常咨询到专业实验协议生成的全方位服务。

## 🔄 整体流程架构

```
用户输入 → 意图识别 → 智能路由 → 专业处理 → 结果输出
    ↓         ↓         ↓         ↓         ↓
  用户交互   分析判断   流程分发   任务执行   反馈结果
```

## 📋 详细流程说明

### 第一阶段：用户输入接收
**目标**: 接收并预处理用户的自然语言输入

**处理步骤**:
1. **输入接收**: 接收用户的文本输入
2. **输入预处理**:
   - 文本清洗和格式化
   - 关键词提取
   - 上下文关联分析
3. **初步分类**: 根据输入特征进行初步分类

**关键功能**:
- 支持多语言输入
- 处理模糊表达
- 上下文记忆

### 第二阶段：意图识别
**目标**: 准确识别用户的真实意图类型

**识别分类**:
```
A. 日常闲聊 (Normal Chat)
   - 问候咨询
   - 功能介绍
   - 系统使用指导
   - 日期时间查询

B. 希望结束聊天 (End Chat)
   - 用户终止对话
   - 结束服务请求

C. Opentrons设备控制意图 (Device Control)
   - 设备启动/停止
   - 状态查询
   - 实时控制操作
   - 调试和维护

D. Opentrons代码生成意图 (Code Generation)
   - 实验协议开发
   - Python代码生成
   - SOP标准化流程
   - 模拟验证
```

**识别算法**:
```python
def detect_user_intent(user_input: str) -> str:
    # 1. 结束聊天关键词检测（最高优先级）
    if end_chat_score > 0:
        return "end_chat"
    # 2. 日期时间问题一律归为日常闲聊
    elif contains_time_keywords(user_input):
        return "normal_chat"
    # 3. 设备控制关键词判断
    elif device_control_score > 0:
        return "device_control"
    # 4. 实验相关关键词判断
    elif code_score > 0:
        return "code_generation"
    # 5. 日常闲聊判断
    elif normal_score > 0 or len(user_input) < 10:
        return "normal_chat"
    # 6. 不明确情况优先归日常闲聊
    else:
        return "normal_chat"
```

### 第三阶段：智能路由
**目标**: 根据意图识别结果，路由到对应的专业处理模块

**路由规则**:
```
意图类型 → 处理模块 → 响应模板
    ↓          ↓         ↓

日常闲聊 → 日常对话模块 → 系统介绍和指导
结束聊天 → 对话结束模块 → 礼貌结束服务
设备控制 → 设备控制模块 → MCP协议设备操作
代码生成 → 协议生成模块 → 实验协议开发流程
```

**路由决策逻辑**:
- **意图匹配**: 基于关键词和语义分析
- **上下文考虑**: 考虑对话历史和状态
- **优先级处理**: 按照优先级处理不同意图
- **错误处理**: 处理识别错误和模糊情况

### 第四阶段：专业处理
**目标**: 根据路由结果，执行对应的专业处理流程

#### 4.1 日常对话处理流程
```
用户咨询 → 系统功能介绍 → 使用指导 → 问题解答
    ↓         ↓           ↓         ↓
  接收问题   提供介绍     给出建议   解决疑问
```

**核心功能**:
- 系统功能介绍
- 使用方法指导
- 常见问题解答
- 用户需求澄清

#### 4.2 设备控制处理流程
```
设备控制指令 → 设备类型识别 → MCP协议连接 → 设备操作执行 → 结果反馈
      ↓            ↓            ↓            ↓           ↓
   指令解析       OT-2/Flex    建立连接     执行操作     状态返回
```

**核心功能**:
- 设备类型识别（OT-2/Flex）
- MCP协议通信
- 实时设备控制
- 状态监控和反馈

**设备类型识别算法**:
```python
def identify_device_type(hardware_config: str) -> dict:
    # 分析硬件配置特征
    if contains_flex_features(hardware_config):
        return {"device_type": "flex", "confidence": 0.9}
    elif contains_ot2_features(hardware_config):
        return {"device_type": "ot-2", "confidence": 0.9}
    else:
        return {"device_type": "unclear", "confidence": 0.3}
```

#### 4.3 代码生成处理流程
```
实验需求 → 硬件配置分析 → SOP生成 → 代码开发 → 模拟验证 → 迭代优化
    ↓         ↓           ↓        ↓        ↓         ↓
  需求分析   设备兼容性   流程设计   Python   错误检查   修复完善
```

**核心功能**:
- 实验需求分析
- 硬件配置验证
- SOP标准化生成
- Python代码开发
- 模拟验证和调试
- 错误修复和优化

**代码生成步骤**:
1. **需求分析**: 理解用户实验需求
2. **硬件配置**: 确认设备和耗材配置
3. **SOP设计**: 生成标准化操作流程
4. **代码开发**: 编写可执行Python代码
5. **模拟验证**: 运行代码模拟验证
6. **迭代优化**: 根据错误进行修复优化

### 第五阶段：结果输出
**目标**: 向用户提供清晰、专业的处理结果

**输出格式**:
- **结构化结果**: 清晰的层次结构
- **专业术语**: 使用领域专业术语
- **操作指导**: 提供具体的操作指导
- **错误处理**: 提供错误解决方案

## 🔍 特殊处理流程

### 5.1 代码迭代优化流程
当代码生成失败时，启动迭代优化流程：
```
错误检测 → 错误分析 → 修复策略 → 代码修复 → 验证确认
    ↓        ↓         ↓         ↓         ↓
  识别错误   分析原因   制定方案   实施修复   确认效果
```

**错误类型处理**:
- **语法错误**: 立即修复，最高优先级
- **API错误**: 根据设备类型调整API调用
- **硬件错误**: 修正硬件配置和定义
- **逻辑错误**: 重新设计执行流程

### 5.2 深度错误分析流程
当多次迭代仍无法解决问题时，启动深度分析：
```
历史错误收集 → 关联分析 → 根本原因识别 → 解决方案生成 → 可行性评估
      ↓           ↓          ↓            ↓           ↓
    错误统计     模式识别    问题定位      方案设计     效果预测
```

### 5.3 设备分析后意图匹配流程
```
设备类型识别 → 控制意图匹配 → 专用协议选择 → MCP通信建立 → 操作执行
    ↓           ↓            ↓           ↓           ↓
   OT-2/Flex   控制类型     设备专用     连接设备     实时控制
```
