# LabscriptAI 配置管理指南

## 概述

LabscriptAI 提供了灵活的配置管理系统，支持多种配置方式，让您可以轻松管理API密钥、服务器设置、数据库连接等配置信息。

## 配置优先级

配置系统按以下优先级加载配置：

1. **环境变量** (最高优先级)
2. **config.json 文件**
3. **默认值** (最低优先级)

## 快速开始

### 方法一：使用配置设置脚本 (推荐)

运行交互式配置脚本：

```bash
python scripts/setup_config.py
```

脚本将引导您完成所有必要的配置设置。

### 方法二：手动配置

1. 复制示例文件：
   ```bash
   cp .env.example .env
   cp config.json.example config.json
   ```

2. 编辑配置文件，填入实际值。

## 配置文件详解

### .env 文件

环境变量文件，适合存储敏感信息如API密钥：

```bash
# 应用环境
LABSCRIPTAI_ENVIRONMENT=development

# 主要 LLM API 配置
LABSCRIPTAI_API_KEY=your_openai_api_key_here
LABSCRIPTAI_BASE_URL=https://api.openai.com/v1
LABSCRIPTAI_MODEL_NAME=gpt-4

# DeepSeek API 配置 (用于快速任务)
LABSCRIPTAI_DEEPSEEK_API_KEY=your_deepseek_api_key_here
LABSCRIPTAI_DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
LABSCRIPTAI_DEEPSEEK_MODEL=deepseek-chat

# 服务器配置
LABSCRIPTAI_SERVER_HOST=0.0.0.0
LABSCRIPTAI_SERVER_PORT=8000
LABSCRIPTAI_DEBUG=false

# 数据库配置
LABSCRIPTAI_DATABASE_URL=sqlite:///./labscriptai.db

# 日志配置
LABSCRIPTAI_LOG_LEVEL=INFO
LABSCRIPTAI_LOG_FILE_PATH=logs/labscriptai.log
```

## 环境变量命名规范

所有环境变量都以 `LABSCRIPTAI_` 为前缀，遵循以下命名规范：

- `LABSCRIPTAI_ENVIRONMENT` - 运行环境
- `LABSCRIPTAI_API_*` - API相关配置
- `LABSCRIPTAI_SERVER_*` - 服务器相关配置
- `LABSCRIPTAI_DATABASE_*` - 数据库相关配置
- `LABSCRIPTAI_LOG_*` - 日志相关配置

## API 配置详解

### 支持的 LLM 提供商

#### OpenAI 兼容 API

```bash
LABSCRIPTAI_API_KEY=sk-your-openai-key
LABSCRIPTAI_BASE_URL=https://api.openai.com/v1
LABSCRIPTAI_MODEL_NAME=gpt-4
```

#### DeepSeek API

```bash
LABSCRIPTAI_DEEPSEEK_API_KEY=sk-your-deepseek-key
LABSCRIPTAI_DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
LABSCRIPTAI_DEEPSEEK_MODEL=deepseek-chat
```

#### 其他兼容提供商

系统支持任何 OpenAI 兼容的 API，只需设置相应的 base_url：

```bash
# Azure OpenAI
LABSCRIPTAI_BASE_URL=https://your-resource.openai.azure.com/openai/deployments/your-deployment

# 本地部署的模型
LABSCRIPTAI_BASE_URL=http://localhost:11434/v1

# 其他云服务商
LABSCRIPTAI_BASE_URL=https://api.your-provider.com/v1
```

## 在代码中使用配置

### 基本用法

```python
from backend.config_manager import get_config

# 获取配置实例
config = get_config()

# 使用配置
api_key = config.api.api_key
model_name = config.api.model_name
server_port = config.server.port
```

### 检查配置

```python
from backend.config_manager import get_config

config = get_config()

# 检查是否配置了API密钥
if not config.api.api_key:
    raise ValueError("请设置 LABSCRIPTAI_API_KEY 环境变量")

# 检查是否为生产环境
if config.environment == "production":
    # 生产环境特定逻辑
    pass
```

### 动态配置更新

```python
from backend.config_manager import get_config, reload_config

# 重新加载配置（当配置文件发生变化时）
reload_config()
config = get_config()
```

## Docker 部署配置

### 使用环境变量

```yaml
# docker-compose.yml
version: '3.8'
services:
  labscriptai:
    build: .
    environment:
      - LABSCRIPTAI_API_KEY=${LABSCRIPTAI_API_KEY}
      - LABSCRIPTAI_BASE_URL=${LABSCRIPTAI_BASE_URL}
      - LABSCRIPTAI_DATABASE_URL=postgresql://postgres:password@db:5432/labscriptai
    env_file:
      - .env
```

### 使用配置文件挂载

```yaml
# docker-compose.yml
version: '3.8'
services:
  labscriptai:
    build: .
    volumes:
      - ./config.json:/app/config.json:ro
      - ./logs:/app/logs
```

## 安全最佳实践

1. **敏感信息管理**
   - 将 `.env` 文件添加到 `.gitignore`
   - 使用环境变量存储API密钥等敏感信息
   - 在生产环境中使用密钥管理服务

2. **权限控制**
   - 确保配置文件具有适当的文件权限
   - 限制对配置文件的访问

3. **配置验证**
   - 系统会自动验证必需的配置项
   - 在启动时检查配置的有效性

## 故障排除

### 常见问题

1. **配置未生效**
   - 检查环境变量名称是否正确
   - 确认配置文件路径和格式
   - 重启应用程序

2. **API 连接失败**
   - 验证 API 密钥是否正确
   - 检查网络连接和防火墙设置
   - 确认 base_url 是否正确

3. **数据库连接问题**
   - 检查数据库 URL 格式
   - 确认数据库服务是否运行
   - 验证用户名和密码

### 调试配置

启用调试模式查看详细的配置信息：

LABSCRIPTAI_LOG_LEVEL=DEBUG

## 迁移指南

### 从旧配置迁移

如果您之前使用硬编码配置，请按以下步骤迁移：

1. 运行配置设置脚本：
   ```bash
   python scripts/setup_config.py
   ```

2. 将现有的配置值填入新的配置文件

3. 更新代码以使用新的配置管理器：
   ```python
   # 旧方式
   API_KEY = "hardcoded_key"
   
   # 新方式
   from backend.config_manager import get_config
   config = get_config()
   API_KEY = config.api.api_key
   ```

4. 测试应用程序确保一切正常工作
